## VE Team Session Duties

For the SJCARS VE team, the running of a session is broken down into two roles. The ‘Admin’ role is the team leader and manages end to end operation of the session, including running the Session Manager software that produces the 605 and CSCE’s, along with the other artifacts. The Proctor role is focused on actual test taking activities, including all testers interactions and overseeing the actual exam. Both are critical to a successful exam. 

In addition to outlining the specific duties of each role, the document also highlights the overlap or critical interactions that occur between the two roles. Typically a VE session will have at least four VE’s, with the team lead in the Admin role and three Proctors. For small sessions with a handful of testers, three VEs can work and the Team Lead plays the part of both Admin and Proctor as necessary.

## Admin Role Duties

* Prior to session  
  * Collect Sign-up and Felony Form, make sure includes FRN  
  * Enter all data into SessionManager software  
  * Ensure testing box is ready / stocked (enough tests, answer sheets, etc)  
* Day of Session  
  * Print VE sign in sheet, have all VE’s sign  
  * Setup laptop and printer (test)  
  * Pull out and sort tests, answer sheets and misc (pencils, scratch paper)  
* When first tester has completed and receive test/answer sheet from Proctor  
  * Confirm answer sheet was signed by VE’s (if you are the 3rd VE, sign off on test)  
  * As Proctor if tester going for the next level?  
  * When tester done taking test(s), enter their activities in SessionManager  
    * Enter each test taken status (Pass/Fail)  
    * Print out 605/CSCE as necessary  
    * Sign if you are the 3rd VE  
    * Put all answer sheets and sign-in form (copy of license if available) in the ‘Session Folder)   
    * Hand CSCE/605 to Proctor   
* When Proctor returns 605  
  * Make sure student signed everything  
  * Marked as completed and returned form in SessionManager for student (this has to be done for all testers before you can close session)  
  * Put 605 in Session Folder in front of testers answer sheets and sign-in form  
* When all testers have returned 605  
  * Close out session  
  * Generate a ENC file for the session. This is the file that Kenny uploads to the FCC. File will be named something like “BYSJCARS220622A0.enc” \=  
  * Print Session Report  
  * Pull out all papers from Session Folder, organize in pile in following order:  
    * Session report  
    * VE sign in sheet  
    * 605 for each tester, followed by their answer sheets/scratch paper and sign in form  
    * Answer sheet of anybody who didn’t pass (so no 605\) and their scratch paper/sign in form  
  * Generate PDF of all the paper artifacts using mobile app (I use Genius Scan)   
  * Send kenny PDF and ENC file  
* Pack up Testing box  
  * Note any tests that need to be replaced, answer sheets needed, etc.

## Proctor Role

* Prior to Session  
  * Confirm location/start time  
  * Make sure ZipGrade installed on mobile device and working  
    * If app hasn’t been logged into team account, contact Kenny  
* Day of Session  
  * Meet each tester and get them seated (ensure separated from other testers)  
  * Share bathroom location and remind if leave during the test, the test is over.  
  * Review / confirm ID’s match their sign-up form (if testing General/Extra, review copy of license)  
  * Give tester the testing material (Test, answer sheet and writing tool if needed) from admin table/bin  
    * Make sure they put the test number on answer sheet  
    * Ask to write name on scratch paper  
    * Ask tester to NOT WRITE on test  
    * Mention that diagrams are on the back page   
    * Ensure if they have any questions, please ask  
    * If they need a calculator, please request one  
    * No phones, tablets or other electronic devices can be out during test.  
    * Tell them good luck\!  
  * Remind room that once the first test starts, to please limit talking and noise making  
  * When tester is done, collect their materials  
    * Double check name on answer sheet  
    * Double check that answer sheet has correct test number  
    * Take test and put back in stack of tests, after flipping through to check for marks  
    * Hand test to first VE to grade (ZipGrade)  
      * Sign answer sheet when putting score  
        * MUST mark PASS or FAIL  
        * MUST fill in number of right and wrong  
        * MUST put callsign on the answer sheet  
      * Remember ZipGrade is setup for 50 questions, but Tech/Gen is only 35, so you need to subtract the number missed from 35\.  
      * Hand to second VE Grader (repeat above)  
    * When test graded, if 3rd VE is available (that isn’t admin), have them review the work of first two and sign. Only two VE need to ‘grade’ if using ZipGrader.   
      * MUST put the callsign on the answer sheet, if missing either of other proctors, take it back to them.  
    * Take the answer sheet to Admin. Can tell the tester if passed at this point. HIGHLY RECOMMEND don’t go down the road of showing them which questions missed. You can tell the score (missed 3 or something), but this isn’t a ‘teaching’ time.  
      * If a tester passed, ask if they want to take the next test.  
      * If the tester didn’t pass, ask if they would like to take the test again. Consider how many they missed when pushing this option. If they took Element 4 and missed 25%, they won’t pass on a different test.  
      * If testing again, go back to the top.  
    * When Admin has completed processing a passed test, collect the CSCE and 605 for the tester to sign.  
      * Make sure all the VE’s have signed both forms before taking to the tester  
      * Make sure tester signs ALL the forms before giving them the CSCE  
        * Make sure to have them double check address and e-mail  
        * Remind them they have to pay for their first license before it is issued. And their e-mail has to be correct.  
        * DO NOT GIVE THEM THE CSCE UNTIL THEY HAVE SIGNED THE 605\!\!\!\!\!  
      * Bring back all the signed forms to the Admin.  
      * Congratulate tester and guide them out of testing space (if others are still testing)

## FAQ

**What happens if no testers show up?**

* Still need to generate the following files, even if no tests were given, to get credit for the session.  
  * SessionSummary (will say zero tests given and zero upgrades)  
  * VESignIn   
  * Empty or blank .enc file (generate like normal, will complain that empty)

**What happens if no tester passes?**

* Treat like a normal session, except no 605 or CSCE are generated.  
* Still need to generate Session Summary, VE Sign and ENC (blank) file and PDF with all the testing artifacts (answer sheets, etc)
